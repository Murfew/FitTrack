## Design Context

### Users

Solo athlete — the builder themselves. Two distinct usage contexts:

- **In the gym (mobile)**: Fast logging of sets, reps, weights mid-session. Interface must be glanceable and low-friction. Thumbs only. No time to navigate menus.
- **At home (desktop)**: Reviewing progress, analyzing trends, logging food and daily metrics. More space, more patience, more data density acceptable.

The user is their own audience. There are no onboarding flows to design for new users, no social features, no sharing. Every decision optimizes for the person who already knows how the app works.

### Brand Personality

**Three words: precise, deliberate, data-rich**

FitTrack should feel like a well-made precision instrument — a high-quality stopwatch, a serious athlete's training log, Garmin telemetry. Not a wellness app. Not a social platform. Not gamified.

Quiet confidence. The interface recedes so the numbers come forward.

Reference: RepCount (clean, focused, numbers-first) — but more premium, more design-intentional.
Anti-references: MyFitnessPal (overcrowded, cluttered, dated), Hevy (social noise, bro-ish).

### Aesthetic Direction

**Theme**: Dark. Not dark-with-glowing-accents. Dark with restraint — zinc-950 deep background, surfaces that barely lift from the floor, hierarchy through lightness not decoration.

**"Light"** does not mean light mode here. It means *uncluttered and purposeful*. Every element earns its space. If something can be removed without losing meaning, remove it.

**Accent color**: Deep cobalt — `oklch(0.52 0.18 255)`. Precision instrument, aerospace, Polar/Suunto wearables. Not cyan, not indigo — sits in the blue family but reads as mechanical and deliberate. The user's preferred color, executed with restraint on dark.

**Typography**:
- Display/headings: **Barlow Condensed** — tight, purposeful, athletic without aggression. Load via `next/font/google`.
- Body/labels/data: **Figtree** — clean, warm, excellent number rendering. Load via `next/font/google`.
- Monospace for raw numeric data (weights, reps, distances): retain Geist Mono or switch to a matching mono face.
- Numbers and metrics get visual prominence — larger, heavier, monospaced where precision demands.

**Colors** (OKLCH, dark mode primary):
- Background: `oklch(0.13 0.005 255)` — near-black with the faintest cool tint toward cobalt
- Card surface: `oklch(0.19 0.005 255)` — barely lifted, same cool tint
- Border: `oklch(0.28 0.005 255 / 60%)` — subtle, not prominent
- Foreground (primary text): `oklch(0.96 0.006 255)` — cool white, never pure white
- Muted text: `oklch(0.58 0.01 255)` — same hue family, tinted neutral
- Accent: `oklch(0.52 0.18 255)` — deep cobalt, used sparingly
- Accent hover: `oklch(0.60 0.17 255)` — slightly lighter
- Accent foreground: `oklch(0.96 0.006 255)` — light text on dark cobalt button
- Destructive: red-leaning warm tone `oklch(0.65 0.22 25)`
- Success: muted green `oklch(0.60 0.14 155)`

**Spacing and layout**: Generous whitespace on desktop. Tight but clear on mobile. Data density is high but breathing room is built in through hierarchy, not reduction.

### Design Principles

1. **Numbers are the hero.** Weights, reps, streaks, PRs — these get visual priority. Chrome (borders, backgrounds, labels) supports them but never competes.

2. **Context-first layout.** The gym view (mobile) and the review view (desktop) have different information needs. Don't shrink desktop UI to fit mobile — design each context for its purpose.

3. **Restraint is the standard.** If it doesn't carry information or guide the eye, it doesn't belong. No decorative gradients, no card-for-every-item, no icons that restate the text next to them.

4. **Precision in every detail.** Number alignment, unit placement, spacing between data points — all of it should feel considered. Sloppy data display undermines the entire product.

5. **Dark without drama.** The dark palette should feel like a well-lit gym at 5am — functional, focused, no mood lighting. Not a gaming UI.
