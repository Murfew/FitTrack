name: FitTrack planned tracking scope
description: The four data domains FitTrack will track — relevant when designing features, schemas, navigation, and empty states
type: project
---

FitTrack will track four domains: **workouts**, **nutrition**, **steps**, and **body weight (BW)**.

**Why:** This is the intended product scope beyond the current workout-only Phase 1 implementation.

**How to apply:** When designing schemas, navigation, icons, empty states, or any feature that implies the full product surface, account for all four domains — not just workouts. Avoid naming or structuring things in ways that would need to be reworked when nutrition/steps/BW land.
